# 一、项目背景

这是一个财经投资研究类 App，现有技术栈大致为：

* 后端：FastAPI
* 数据源：当前主要使用 yfinance 或类似免费行情源
* 前端：Vue 3 + Vite
* UI：Element Plus
* 图表：ECharts

产品定位是面向个人投资者、研究型用户和金融学习用户的投资研究工具，核心能力包括：

1. 市场行情查看
2. 资产搜索与资产详情
3. K 线和基础指标
4. 多资产对比
5. 组合回测与组合分析
6. 名人组合或模板组合复现
7. 虚拟账户 / 模拟交易
8. 后续预留 AI 分析、新闻事件、投资报告等能力

当前 MVP 已经能跑通部分功能，但还存在明显问题：

1. 界面不够美观，产品感不足。
2. 页面布局、交互路径和视觉风格不统一。
3. 一些功能板块可能存在 bug、边界情况处理不完整、错误态不清晰。
4. 数据展示可信度不足，例如数据源、更新时间、缺失数据、复权方式等提示不充分。
5. 用户完成核心任务的路径还不够顺畅。
6. 产品目前更像技术 Demo，而不是一个打磨过的金融研究工具。

# 七、功能稳定性与数据可信度任务

## 7.1 API 返回格式统一

```json
{
  "success": true,
  "data": {},
  "meta": {
    "source": "yfinance",

    "timezone": "America/New_York",

    "warnings": []
  },
```

错误时：

```json
{
  "success": false,
  "data": null,
  "meta": {},
  "error": {
    "code": "INVALID_SYMBOL",

}
```

前端必须基于该结构处理：
